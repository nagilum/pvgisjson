PVGIS API

Photovoltaic Geographical Information System (PVGIS) API

Solar energy is one of the environmentally sustainable resources for producing electricity using photovoltaic (PV) systems. The main input data used in the planning process is solar radiation. The Europe JRC has an app online which you can use to calculate PV values, bus sadly they have no (good) API. That's where this website comes into play.

The PVGIS API provides a simple API endpoint which you can get the same values via JSON. The API and source code is freely available to use under the MIT license.

How to use

Do a POST request to https://pvgisjson.com/api/v1/pv. 

POST https://pvgisjson.com/api/v1/pv

Include the following JSON payload (lat and lng) as a minimum: 

{
    "lat": 63.3622086,
    "lng": 10.3907399
}

Customize

Here are a list of all 8 attributes you can include in the API request.

PV Technology

The performance of PV modules depends on the temperature and on the solar irradiance, but the exact dependence varies between different types of PV modules. 

{
    /* You can either use "crystSi", "CIS", or "CdTe" and the "pvtech" value. */
    /* The default value is "crystSi".
    "pvtech": "crystSi"
}

Installed peak power

This is the power that the manufacturer declares that the PV array can produce under standard test conditions, which are a constant 1000W of solar irradiation per square meter in the plane of the array, at an array temperature of 25째C. 

{
    /* The default value is 1. */
    "peakpower": 1
}

System loss

The estimated system losses are all the losses in the system, which cause the power actually delivered to the electricity grid to be lower than the power produced by the PV modules. 

{
    /* The default value is 14. */
    "losses": 14
}

Mounting position

For fixed (non-tracking) systems, the way the modules are mounted will have an influence on the temperature of the module, which in turn affects the efficiency. 

{
    /* You can either use "free" or "building". */
    /* The default value is "free". */
    "mounting": "free"
}

Slope

This is the angle of the PV modules from the horizontal plane, for a fixed (non-tracking) mounting. 

{
    /* The default value is 35 */
    "slope": 35
}

Azimuth

The azimuth, or orientation, is the angle of the PV modules relative to the direction due South. -90째 is East, 0째 is South and 90째 is West. 

{
    /* The default value is 0. */
    "azimuth": 0
}
